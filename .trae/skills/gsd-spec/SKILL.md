# GSD Spec Skill

GSD (Get Shit Done) methodology follow kore clear specification build kora ebong feature implement korar jonno ei skill.

---

## CRITICAL: GSD Philosophy

1.  **No Enterprise Theater**: Sprint ceremonies, story points, ba roleplay dorkar nai. Sudhu kaj shesh kora main goal.
2.  **Clear Specs**: Build korar age thik koren "What exactly do I want?". Specs clear thakle code consistent hobe.
3.  **Banglish**: Narration ebong explanation Bengali language e English script e bolben.

---

## Spec Creation Workflow

1.  **Extract Requirements**: User er kotha theke core requirements ber koren.
2.  **Define Specs**: Technical details, logic flow, ebong data structure define koren.
3.  **Verification Steps**: Kibhabe verify kora hobe (tests ba shell commands) sheta plan koren.

---

## Final Goal

Ekta solid spec toiri kora jate implement korar shomoy kono ambiguity na thake.
